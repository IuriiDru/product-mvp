# product-mvp — Юнивеж

Работающее приложение (MVP), **не лендинг**. Лендинг — отдельный проект (`/root/Downloads/landing.html`).

## Основа
Продуктовые требования: [`/root/Downloads/PRD.md`](/root/Downloads/PRD.md)
Фокусный сегмент: оптово-производственные компании на ОСНО с НДС и ВЭД.

## Что делает приложение (предлагаемая концепция MVP)
**«Экспресс-аудит налоговых рисков» — карта налоговых рисков за пару минут.**
Это «работающая» реализация лид-магнита и aha-момента из PRD («Вы получаете карту
налоговых рисков вашей компании на одной странице»).

Пользователь (собственник / финдиректор):
1. Проходит короткий опрос о компании — налоговый режим (ОСНО/УСН), НДС, ВЭД,
   примерный оборот, кто ведёт учёт, были ли проверки/штрафы/блокировки, сдают ли в срок.
2. Получает **персональную карту налоговых рисков**: общий уровень риска + разбивка по
   категориям (НДС, налог на прибыль, ВЭД/валютный контроль, проверки ФНС, штрафы/блокировки),
   с понятными пояснениями и рекомендациями.
3. Видит CTA «Получить бесплатный полный аудит» → оставляет контакт (имя, телефон, компания) →
   заявка сохраняется как лид.

### Ключевые фичи MVP
- Пошаговый опросник (wizard).
- Движок оценки рисков (правила на основе ответов).
- Страница результата: общий скоринг + по категориям + рекомендации.
- Захват лида в базу (PostgreSQL).
- Простая страница «Заявки» для просмотра лидов (админ).

### Зачем это бизнесу (из PRD)
- Реализует aha-момент и точку входа воронки → привлечение маржинальных клиентов.
- Даёт ценность ещё до продажи (валидация рисков №1 «премия за качество» и №2 «канал»).

## Что уже реализовано (MVP)
- `/` — главная: интро + запуск аудита
- `/audit` — пошаговый опросник (10 вопросов) → карта рисков по 5 категориям → форма заявки
- `/api/leads` — сохранение заявки в БД (POST)
- `/admin` — список собранных заявок (без пароля)
- Движок скоринга: `lib/audit.ts`

## Как запустить локально
```bash
npm install          # postinstall сам сгенерирует Prisma Client
npm run db:push      # создать схему в SQLite (dev.db)
npm run dev          # http://localhost:3000
```

## Деплой на PostgreSQL (VPS)
1. В `prisma/schema.prisma` сменить `provider = "sqlite"` → `"postgresql"`.
2. В `.env` указать строку подключения PostgreSQL в `DATABASE_URL`.
3. `npm run db:push` (или `prisma migrate deploy`) → `npm run build` → `npm start`.
